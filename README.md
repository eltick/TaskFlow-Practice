# TaskFlow Practice

TaskFlow Practice - учебная канбан-доска для задач. Проект начинался как обычный todo, но был расширен до доски с пользовательскими колонками, drag-and-drop и карточками с подробным описанием.

Каталожная основа: [React Todo App Tutorial](https://www.youtube.com/watch?v=hT3j87FMR6M)

## Возможности

- создать задачу с названием, описанием, сроком, тегами и приоритетом;
- создать новую колонку;
- удалить колонку и перенести задачи в оставшуюся колонку;
- переносить задачи между колонками перетаскиванием;
- открыть карточку в модальном окне;
- скрывать длинное описание внутри карточки;
- искать и фильтровать задачи;
- хранить задачи и колонки локально или в Supabase.

## Стек

Frontend:

- React;
- TypeScript;
- Vite;
- CSS.

Хранение данных:

- localStorage для локального режима;
- Supabase REST API и PostgreSQL при заполненном `.env`.

В Supabase используются таблицы:

- `tasks` - задачи, статусы, сроки, теги и приоритеты;
- `task_columns` - пользовательские колонки доски.

## Как запустить

```bash
npm install
npm run dev
```

Для подключения Supabase создайте `.env`:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

После изменения `.env` нужно полностью перезапустить dev-сервер.

SQL-схема находится в `supabase/schema.sql`. Ее нужно выполнить в Supabase SQL Editor. Скрипт создает таблицы `tasks` и `task_columns`, а также добавляет стартовые колонки: "К выполнению", "В работе", "Готово".

Если `.env` не заполнен, проект работает через localStorage.

## Структура проекта

- `src/app` - точка входа приложения;
- `src/pages/board` - экран доски и состояние;
- `src/widgets` - шапка, сайдбар и канбан-доска;
